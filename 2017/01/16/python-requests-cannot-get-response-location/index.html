python requests cannot get response location

Date 2017-01-16 By Yin Tags python / requests / scrapy

最近想通过python的urllib库来下载一些文件，但是结果不如愿，因为下载下来的文件并不是想要的，看看原因发现，网页在过程中发生了重定向(状态码为302)

In [1]:
from IPython.display import Image
Image('icons/web_log.png')
Out[1]:

通过网页日志可以看到(Preserve log打勾)，第一个进入的网页的状态码为200。通过python的requests库得到的内容也就截至到这个网页。

从第二个网页中可以看到，状态码变为了302

In [2]:
Image('icons/web_log_302.png')
Out[2]:

那么问题就来了，如何让程序跳转到第二个url中。

网上找了很多方法：

  1. 通过history()方法可以看到历史的跳转记录，但是我看到的为空，因为我得到的网页根本不需要跳转，它的状态码确实为200。

  2. 这篇文章说要用ghost.py解决。

In [2]:
import requests
session = requests.Session()
url = (u"http://ggjd.cnstock.com/ggjd/download/2000684852.PDF")
In [3]:
req = session.get(url)
In [4]:
print req.headers
print req.url

{'X-Via': '1.1 iazai238:0 (Cdn Cache Server V2.0)', 'X-Powered-By': 'PHP/5.3.25', 'Transfer-Encoding': 'chunked', 'Content-Encoding': 'gzip', 'Server': 'nginx', 'Connection': 'keep-alive', 'Date': 'Tue, 17 Jan 2017 13:20:29 GMT', 'Content-Type': 'text/html; charset=utf-8'}
http://ggjd.cnstock.com/ggjd/download/2000684852.PDF

Image('icons/web_log_detail.png')

将(Use small request row)选项打开之后，发现第二个页面的url是发生了改变的。之前为ggjd，现在为ggdl，豁然开朗。将url中的相关字段改了之后，发现可以下载了。并且之后得到的url也是重定向之后的。

In [5]:
url1 = u"http://ggjd.cnstock.com/ggdl/download/2000684852.PDF"
session = requests.Session()
req = session.get(url1)
print req.history
print req.url
