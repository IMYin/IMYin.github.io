特征提取与处理：从图片中提取特征

Date 2016-09-24 By Yin Tags 兴趣点 / SIFT / SURF

上篇文章中我们介绍了如何从文档中提取特征，在工作中还经常会遇到处理图片的问题，比如通过图片识别图片中花的种类。这篇文章我就来讲一下如何从图片中提取特征向量。

通过像素提取特征值¶

数字图像通常是一张光栅图或像素图，将颜色映射到网格坐标里，所以一张图片可以看成是有颜色值构成的矩阵。光学文字识别(Optical character recognition)是机器学习中一个经典的问题。下面我们来举个例子。

scikit-learn中的digits包含了1700多张从0到1的手写数字灰度图片。每张有8x8像素构成，每个像素用0-16数字表示，比如白色是0，黑色是16。

形成特征向量之后我们就可以处理一些基本任务，比如识别手写字母。但是在处理大图像的时候就会使得维度很高，并且和TF-IDF特征向量不同，大部分图像都不是稀疏的，这种表示法的缺点不只是特征向量的维度灾难，如果某个位置的学习结果再经过图像的缩放、旋转或者变换之后可能就不对了，非常敏感。另外，这种方法对图像的亮度也非常敏感。现在计算机视觉应用通常手工实现特征提取，或者用深度学习自动化解决无监督问题。

对兴趣点进行特征提取¶

我们之前创建的特征向量包含了图片中的每个像素，所以有一些噪声也包含了进去。在查看图像之后，我们发现每个图像都有一个白边，这些像素是没用的。我们人类在观察物体时，不需要知道物体中的所有特征，当我们看到某个事物的轮廓就可以识别出来，比如看到一张嘴就可以知道这是不是一个人。我们称这些可以给我们带来大量信息的属性为兴趣点(points of interest)，由丰富的纹理包围，基本可以重建图像。边缘(edges)和角点(corners)是两种常用的兴趣点类型。边是像素快速变化的分界线(boundary)，角是两条边的交集。

In [3]:
import numpy as np
from skimage.feature import corner_harris,corner_peaks
from skimage.color import rgb2gray
import skimage.io as io
from skimage.exposure import equalize_hist

def show_corners(corners,image):
    fig = plt.figure()
    plt.gray()
    plt.imshow(image)
    y_corner,x_corner = zip(*corners)
    plt.plot(x_corner,y_corner,'or')
    plt.xlim(0,image.shape[1])
    plt.ylim(image.shape[0],0)
    fig.set_size_inches(np.array(fig.get_size_inches()) * 1.5)
    plt.show()
    
mandrill = io.imread('icons/mandrill.png')
mandrill = equalize_hist(rgb2gray(mandrill))
corners = corner_peaks(corner_harris(mandrill),min_distance=2)
show_corners(corners,mandrill)

可以看到，兴趣点已经被提取。图中有230400个像素，466个兴趣点被提取。这种方法在提取兴趣点时，即使图片的亮度整体发生变化时，兴趣点依然存在。不过在运行时，上面用到的几个方法需要通过help好好琢磨一下。

SIFT and SURF¶

尺度不变特征转换(Scale-Invariant Feature Transform，SIFT)是一种特征提取方法，相比前面使用的方法，SIFT对图像的尺寸，旋转，亮度变化更不敏感。每个SIFT特征都是一个描述图片上某个区域边缘和角点的向量。和兴趣点不同，SIFT还可以获取每个兴趣点和它周围点的综合信息。加速稳健特征(Speeded-Up Robust Features，SURF)是另一个抽取图像兴趣点的方法，其特征向量对图像的尺寸，旋转，亮度变化是不变的。SURF的算法可以比SIFT更快，更有效的识别出兴趣点。

和抽取兴趣点一样，抽取SURF只是机器学习中创建特征向量的第一步，训练集的每个实例都会抽取不同的SURF。这里需要用到mahotas库。

数据标准化¶

许多评估方法在处理标准化数据集时可以获得更好的效果。标准化数据均值为0，单位方差(Unit Variance)。均值为0的解释变量是关于原点对称的，特征向量的单位方差表示其特征值全身统一单位，统一量级的数据。例如，假设特征向量由两个解释变量构成，第一个变量值范围[0,1]，第二个变量值范围[0,1000000]，这时就要把第二个变量的值调整为[0,1]，这样才能保证数据是单位方差。如果变量特征值的量级比其他特征值的方差还大，这个特征值就会主导学习算法的方向，导致其他变量的影响被忽略。有些机器学习算法会在数据不标准时引入很小的优化参数值。解释变量的值可以通过正态分布进行标准化，减去均值后除以标准差。scikit-learn的scale函数可以实现：

from sklearn import preprocessing
import numpy as np
X = np.array([
    [0., 0., 5., 13., 9., 1.],
    [0., 0., 13., 15., 10., 15.],
    [0., 3., 15., 2., 0., 11.]
])
print(preprocessing.scale(X))

总结¶

我们介绍了图像特征提取的方法。首先，我们介绍了一个关于的手写数字识别的OCR问题，通过图像的像素矩阵扁平化来学习手写数字特征。这种方法非常耗费资源，于是我们引入兴趣点提取方法，通过SIFT和SURF进行优化。最后介绍了数据标准化的方法，确保解释变量的数据都是同一量级，均值为0的标准化数据。
