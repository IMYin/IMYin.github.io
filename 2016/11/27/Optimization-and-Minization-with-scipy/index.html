Scipy ---> Optimization and Minimization

Date 2016-11-27 By Yin Tags python / numpy / scipy / intersection / function / roots

With Numpy we can achieve fast solutions with simple code. Where does Scipy come into the picture? It's a package that utilizes Numpy arrays and manipulations to take on standard problems that scientists and engineers commonly face: integration(积分), determining a function's maxima or minima, finding eigenvectors(特征向量) for large sparse matrices, testing whether two distributions are the same, and much more. We will cover just the basics here, which will allow you to take advantage of the more complex features in Scipy by going through easy examples that are applicable to real-world problems.

We will start with optimization and data fitting, as these are some of the most common tasks, and then move through interpolation(插值), integration, spatial anaysis(空间分析), clustering, signal(信号) and image processing, sparse matrices, and statistics.

Optimization and Minimization¶

The optimization package in Scipy allows us to solve minimization problem easily and quickly. But wait: what is minimization and how can it help you with your work? Some classic example are performing linear regression, finding a function's minimum and maximum values, determining the root of a function, and finding where two functions intersect(相交). But we begin with a simple linear regression and then expand it to fitting non-linear data.

Attention¶

The optimization and minimization tools that Numpy and Scipy provide are great, but they do not have Markov Chain Monte Carlo(MCMC) capabilities --> in other words, Bayesian analysis. There are several popular MCMC Python packages like PyMC, a rich package with many options, and emcee, an affine(仿射) invariant(恒定的) MCMC ensemble sampler (meaning that large scales are not a problem for it).

Data Modeling and Fitting¶

There are several ways to fit data with a linear regression. In this section we will use curve_fit, which is a $χ^2-based$ method(in other words, a best-fit method). In the example below, we generate data from a known function with noise, and then fit the noisy data with curve_fit. The function we will model in the example in the example is a simple linear qeuation, $f(x) = ax + b$.

In [1]:
import numpy as np
from scipy.optimize import curve_fit

# Creating a function to model and create data
def func(x, a, b):
    return a * x + b

# Generating clean data
x = np.linspace(0, 10, 100)
y = func(x, 1, 2)

# Adding noise to the data
yn = y + 0.9 * np.random.normal(size=len(x))

# Executing curve_fit on noisy data
popt, pcov = curve_fit(func, x, yn)

# popt returns the best fit values for parameters of 
# the given model(func).

print(popt)

[ 0.95916394  2.27074202]

The values from popt, if a good fit, should be close to the value for the y assignment(分配). You can check the quality of the fit with pcov, where the diagonal(对角线) elements are the variances for each parameter. In the example below, I give a visual illustration of the fit.

In [2]:
%matplotlib inline
import matplotlib.pyplot as plt

num = np.arange(0,10,0.1)
y_fit = x * popt[0] + popt[1]

plt.scatter(num,yn)
plt.plot(num, y_fit, 'y.-', label='Best fit')
plt.plot(num, y, 'r-', lw='2.0', label='Function')
plt.legend(loc='best')
plt.plot()

Taking this a step further, we can do a least-squares fit to a Gaussian profile, a non-linear function: $$a * exp(\frac {-(x-\mu )^2}{2\sigma ^2})$$ Where $a$ is a scalar, $\mu$ is the mean, and $\sigma$ is the standard deviation.

y = func(x, 1, 5, 2)

# Executing curve_fit on noisy data
popt, pcov = curve_fit(func, x, yn)

# popt returns the best-fit values for parameters of the given model(func)
print(popt)

Going one more step, we can fit a one-dimensional dataset with multiple Gaussian profiles. The func is now expanded to include two Gaussian equations with different input variables. This example would be the classic case of fitting line spectra(线谱).

In [5]:
# Two-Gaussian model
def func(x, a0, b0, c0, a1, b1, c1):
    return a0*np.exp(-(x - b0) ** 2/(2 * c0 ** 2)) +  a1 * np.exp(-(x - b1) ** 2/(2 * c1 ** 2))

# Generating clean data
x = np.linspace(0, 20, 200)
y = func(x, 1, 3, 1, -2, 15, 0.5)

# Adding noise to the data
yn = y + 0.2 * np.random.normal(size=len(x))

# Since we are fitting a more complex function,
# providing guesses for the fitting will lead to 
# better results.

guesses = [1, 3, 1, 1, 15, 1]
# Executing curve_fit on noisy data
popt, pcov = curve_fit(func, x, yn, p0=guesses)
print popt

#plot the line spectra
y_fit = func(x, popt[0], popt[1], popt[2], popt[3], popt[4], popt[5])
n = np.arange(0,10,0.05)

plt.scatter(n,yn)
plt.plot(n, y_fit, 'y.-', label='Best fit')
plt.plot(n, y, 'r-', lw='2.0', label='Function')
plt.legend(loc='best')

Solutions to Functions¶

With data modeling and fitting under our belts(区域、带状物), we can move on to finding solutions, such as "What is the root of a function?" or "Where do two functions intersect?" Scipy provides an arsenal(军火库、兵工厂) of tools to do this in the optimize module. We will run through the primary ones in this section.

Let's start simply, by solving for the root of an equation. Here we will use scipy.optimize.fsolve.

Finding the intersection points between two equations is nearly as simple.

In [8]:
from scipy.optimize import fsolve
import numpy as np

# Defining function to simplify intersection solution
def findIntersection(func1, func2, x0):
    return fsolve(lambda x : func1(x) - func2(x), x0)

# Defining functions that will intersect
funky = lambda x : np.cos(x / 5) * np.sin(x / 2)
line = lambda x : 0.01 * x - 0.5

# Defining range and getting solutions on intersection points
x = np.linspace(0, 45, 10000)
result = findIntersection(funky, line, [15,20,30,35,40,45])

# Printing out results for x and y
print(result, line(result))

(array([ 13.40773078,  18.11366128,  31.78330863,  37.0799992 ,
        39.84837786,  43.8258775 ]), array([-0.36592269, -0.31886339, -0.18216691, -0.12920001, -0.10151622,
       -0.06174122]))

As we can see in the figure below, the intersection points are well identified. Keep in mind that the assumptions about where the functions will intersect are important. If these are incorrect, you could get specious results.
